## 2.1 User (Utilisateur personnalisé)
```python
class User(models.Model):
    username = models.CharField(max_length=100, unique=True)
    password = models.CharField(max_length=255)  # Hash SHA256
    role = models.CharField(max_length=20, default='user', choices=[
        ('admin', 'Admin'),
        ('user', 'User')
    ])
    created_at = models.DateTimeField(auto_now_add=True)
    
    class Meta:
        db_table = 'portfolio_users'
    
    def is_authenticated(self):
        return True
    
    def is_staff(self):
        return self.role == 'admin'
```

⚠️ IMPORTANT
- L’authentification utilisée par l’admin **n’utilise plus ce modèle `portfolio.User`**.
- L’admin utilise désormais **l’utilisateur Django** (`django.contrib.auth`, table `auth_user`) + **Token DRF**.
- Le modèle `portfolio.User` peut rester pour d’autres usages, mais il n’est pas la source d’auth.

### 2.2bis Gestion des images (Cloudinary) — sélection “native” de fichier
Le backend est configuré avec Cloudinary :
- `DEFAULT_FILE_STORAGE = 'cloudinary_storage.storage.MediaCloudinaryStorage'`
- donc un `ImageField` est **automatiquement stocké sur Cloudinary**.

➡️ Résultat : le frontend peut utiliser un `<input type="file">` et envoyer un **multipart/form-data**.

#### Payload attendu pour créer un projet avec image
- **Content-Type**: `multipart/form-data`
- Champs texte: `project_name`, `project_description`, `technology_used`, `github_link` (optionnel), `demo_link` (optionnel)
- Fichier: **`project_image`** (un fichier image)

#### Exemple Vue/Vite (Axios + FormData)
```js
import axios from "axios";

const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL, // ex: https://portefolio-backend-v0e0.onrender.com/api
});

api.interceptors.request.use((config) => {
  const token = localStorage.getItem("authToken");
  if (token) config.headers.Authorization = `Token ${token}`;
  return config;
});

export async function createProjectWithImage(form) {
  const fd = new FormData();
  fd.append("project_name", form.project_name);
  fd.append("project_description", form.project_description);
  fd.append("technology_used", form.technology_used);
  if (form.github_link) fd.append("github_link", form.github_link);
  if (form.demo_link) fd.append("demo_link", form.demo_link);
  if (form.project_image_file) fd.append("project_image", form.project_image_file); // File

  // IMPORTANT: ne pas fixer Content-Type manuellement, Axios met le boundary.
  const res = await api.post("/projects/", fd);
  return res.data;
}
```

#### Exemple de composant (sélection fichier)
```vue
<input type="file" accept="image/*" @change="e => (file = e.target.files?.[0] ?? null)" />
```

#### Réponse API
DRF renverra `project_image` sous forme d’URL Cloudinary (ou chemin) dans la réponse du projet.


### 2.2bis Gestion de l’image (IMPORTANT)
Dans ce backend, `project_image` est un **URLField** :
- ✅ Le frontend doit envoyer **une URL** (ex: `https://.../image.jpg`)
- ❌ Ce n’est pas un upload de fichier direct vers Django (pas de `multipart/form-data` attendu sur `/api/projects/`)

#### Recommandation côté frontend
1) Upload l’image vers un service (Cloudinary, Supabase Storage, etc.)
2) Récupérer l’URL publique retournée
3) Envoyer cette URL dans `project_image` lors du `POST /api/projects/`

Payload attendu :
```json
{
  "project_name": "Mon projet",
  "project_description": "Description...",
  "technology_used": "Vue, Django, Supabase",
  "project_image": "https://res.cloudinary.com/<cloud>/image/upload/v123/mon-image.jpg",
  "github_link": "https://github.com/...",
```

## 4.1bis Erreurs de validation (400) lors de la création d’un projet
Quand le frontend fait `POST /api/projects/`, Django REST Framework valide les champs via le serializer.
Si un champ est invalide, l’API renvoie **HTTP 400** avec un objet d’erreurs par champ, par exemple :

```json
{
  "project_name": [
    "Ce champ est obligatoire."
  ]
}
```

Erreurs fréquentes sur `project_name` :
- champ vide / manquant
- **doublon** (le champ est `unique=True` dans le modèle)
- longueur > 200 caractères

Erreurs fréquentes sur `project_image` :
- pas un fichier image
- champ envoyé en JSON au lieu de multipart (ex: tu envoies une string)
- fichier trop gros (selon limites Cloudinary / réseau)

# ===============================================
# 5. SYSTÈME D'AUTHENTIFICATION
# ===============================================

## 5.1 Architecture d'authentification
- **Admin Django**: utilisateur Django standard (`auth_user`) avec `is_staff=True`
- **Token DRF**: `rest_framework.authtoken` (persistant, non-expirant par défaut)
- **Stateless**: le frontend envoie le token dans `Authorization`

## 5.2 Flux d'authentification
1. **Création admin (Render)**: au déploiement, `create_admin.py` crée/maj un admin Django à partir des variables Render :
   - `ADMIN_USERNAME=BradyAdmin`
   - `ADMIN_PASSWORD=Admin@1234`
   - (optionnel) `ADMIN_EMAIL=...`
2. **Connexion**: `POST /api/admin/login/` avec `username/password`
3. **Token**: Token DRF retourné (et réutilisé si déjà existant)
4. **Utilisation**: **obligatoire** sur les endpoints d’écriture (POST/PUT/DELETE)

## 5.3 Sécurité implémentée
- **Hash SHA256**: `hashlib.sha256(password.encode()).hexdigest()`
- **Validation**: Champ password requis dans le serializer
- **Rôle vérifié**: Seuls les `admin` peuvent se connecter
- **Token unique**: UUID4 généré à chaque connexion

## 5.4 Exemple d'utilisation dans Vue.js
```javascript
import axios from "axios";

const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL, // ex: https://portefolio-backend-v0e0.onrender.com/api
  headers: { "Content-Type": "application/json" },
});

// 1) Login admin -> récupère token DRF
export async function adminLogin(credentials) {
  const res = await api.post("/admin/login/", credentials); // {username, password}
  if (res.data?.success && res.data?.token) {
    localStorage.setItem("authToken", res.data.token);
  }
  return res.data;
}

// 2) Injecter automatiquement le token sur les requêtes
api.interceptors.request.use((config) => {
  const token = localStorage.getItem("authToken");
  if (token) {
    // IMPORTANT: format DRF TokenAuth
    config.headers.Authorization = `Token ${token}`;
  }
  return config;
});

// 3) Créer un projet (requiert Authorization: Token ...)
export function createProject(payload) {
  return api.post("/projects/", payload);
}
```

### 5.5 Endpoints qui nécessitent le token
- **Public (pas de token)**:
  - `GET /api/projects/`
  - `GET /api/technologies/`
- **Protégés (token obligatoire)**:
  - `POST/PUT/DELETE /api/projects/`
  - `POST/PUT/DELETE /api/technologies/`

### 5.6 Erreurs fréquentes côté frontend (Token)
- **403 "Vous n'avez pas la permission..."**: token absent/mal formé (tu dois envoyer `Authorization: Token ...`)
- **401**: mauvais `username/password` sur `/api/admin/login/`

## 6.2 Service API (src/services/api.js)
```javascript
import axios from 'axios'

const API_BASE_URL = 'https://portefolio-backend-v0e0.onrender.com/api'

const api = axios.create({
  baseURL: API_BASE_URL,
  headers: {
    'Content-Type': 'application/json',
  },
  withCredentials: false // Pas de cookies Django - token based
})

// Intercepteur pour ajouter le token
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('authToken')
  if (token) {
    // IMPORTANT (Option A / DRF TokenAuthentication)
    config.headers.Authorization = `Token ${token}`
  }
  return config
})

export default {
  // Authentification
  createUser(userData) {
    return api.post('/users/', userData)
  },
  
  adminLogin(credentials) {
    return api.post('/admin/login/', credentials)
  },

  // Projets
  getProjects(page = 1, search = '') {
    const params = new URLSearchParams({
      page,
      ...(search && { search })
    })
    return api.get(`/projects/?${params}`)
  },
  
  getProject(id) {
    return api.get(`/projects/${id}/`)
  },

  createProject(data) {
    return api.post('/projects/', {
      project_name: data.project_name,
      project_description: data.project_description,
      technology_used: data.technology_used,
      project_image: data.project_image,
      github_link: data.github_link,
      demo_link: data.demo_link
    })
  },

  updateProject(id, data) {
    return api.put(`/projects/${id}/`, {
      project_name: data.project_name,
      project_description: data.project_description,
      technology_used: data.technology_used,
      project_image: data.project_image,
      github_link: data.github_link,
      demo_link: data.demo_link
    })
  },

  deleteProject(id) {
    return api.delete(`/projects/${id}/`)
  },
  
  // Technologies  
  getTechnologies() {
    return api.get('/technologies/')
  },

  createTechnology(data) {
    return api.post('/technologies/', {
      nom: data.nom,
      imageTechnologie: data.imageTechnologie
    })
  },

  updateTechnology(id, data) {
    return api.put(`/technologies/${id}/`, {
      nom: data.nom,
      imageTechnologie: data.imageTechnologie
    })
  },

  deleteTechnology(id) {

  }
}
```

## 6.3 Store d'authentification (src/stores/auth.js)
```javascript
import { ref } from 'vue'
import api from '../services/api.js'

export const useAuthStore = () => {
  const user = ref(null)
  const token = ref(localStorage.getItem('authToken'))
  const isAuthenticated = ref(!!token.value)

  const login = async (credentials) => {
    try {
      const response = await api.adminLogin(credentials)
      if (response.data.success) {
        user.value = response.data.user
        token.value = response.data.token
        isAuthenticated.value = true
        localStorage.setItem('authToken', token.value)
        return true
      }
      return false
    } catch (error) {
      console.error('Erreur de connexion:', error)
      return false
    }
  }

  const logout = () => {
    user.value = null
    token.value = null
    isAuthenticated.value = false
    localStorage.removeItem('authToken')
  }

  return {
    user,
    token,
    isAuthenticated,
    login,
    logout
  }
}
```

```vue
<template>
  <div class="min-h-screen flex items-center justify-center bg-obsidian">

          <label class="block text-sm font-medium text-gray-custom mb-2">Mot de passe</label>
          <input 
            v-model="form.password"

            required
            class="w-full px-4 py-3 bg-dark-jungle border border-gray-800 rounded-lg focus:outline-none focus:ring-2 focus:ring-emerald text-off-white"
            placeholder="••••••••"
          >
        </div>

        <div v-if="error" class="bg-red-500/20 border border-red-500 text-red-400 px-4 py-3 rounded-lg">
          {{ error }}
        </div>

        <button 
          type="submit" 
          :disabled="loading"
          class="w-full bg-emerald text-obsidian px-6 py-3 rounded-lg hover:bg-neon-lime transition-all font-semibold disabled:opacity-50"
        >
          {{ loading ? 'Connexion...' : 'Se connecter' }}
        </button>
      </form>
    </div>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { useRouter } from 'vue-router'
import { useAuthStore } from '../stores/auth.js'

const router = useRouter()
const authStore = useAuthStore()

const loading = ref(false)
const error = ref('')
const form = ref({
  username: '',
  password: ''
})

const handleLogin = async () => {
  loading.value = true
  error.value = ''
  
  try {
    const success = await authStore.login(form.value)
    if (success) {
      router.push('/admin')
    } else {
      error.value = 'Identifiants invalides'
    }
  } catch (err) {
    error.value = 'Erreur de connexion'
  } finally {
    loading.value = false
  }
}
</script>
```

# 15. RÉSUMÉ DE L'AUTHENTIFICATION
# ===============================================

## 15.1 Points clés de l'implémentation
- ✅ **Modèle User simplifié**: Pas d'héritage Django
- ✅ **Hash SHA256**: Sécurité des mots de passe
- ✅ **Token UUID4**: Simple et efficace
- ✅ **Stateless**: Pas de session côté serveur
- ✅ **Compatible**: Facile à intégrer avec Vue.js
- ✅ **Postman testé**: Création et connexion fonctionnelles

## 15.2 Flux complet
1. **Création admin**: POST /api/users/ → Hash SHA256 → Sauvegarde
2. **Connexion**: POST /api/admin/login/ → Vérification hash → Token UUID4
3. **Utilisation**: Token dans header Authorization → Accès aux endpoints protégés

## 15.3 Pour Vue.js
- Stocker le token dans localStorage
- Ajouter le token dans les headers Axios
- Gérer l'état d'authentification dans un store
- Rediriger selon le statut auth
